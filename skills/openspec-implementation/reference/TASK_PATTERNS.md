# OpenSpec 任务执行模式

本文件给出与官方 OpenSpec 流程一致的任务执行模式，帮助在实施阶段把 `tasks.md` 转化为可执行的 TodoWrite 计划。

## 核心约束

- `tasks.md` 描述实施顺序与完成项
- 运行时进度使用 TodoWrite 跟踪
- 验证必须贴近对应任务执行
- archive 准备状态应来自 `tasks.md` 的完成情况，而不是额外标记文件

## 推荐任务结构

`tasks.md` 推荐使用 Markdown checklist，并按阶段组织：

```markdown
## 阶段 1：准备
- [ ] 任务 A
- [ ] 任务 B
```

## 执行模式 1：线性任务

适用于有明显依赖链的 change。

1. 将当前 checklist 项标记为 `in_progress`
2. 完成并验证后标记为 `completed`
3. 再开始下一项

## 执行模式 2：任务 + 子步骤映射

适用于单个 checklist 项较大，需要更细粒度追踪。

例如：

- [ ] 完成认证流程改造

可拆成 TodoWrite：

- 完成认证流程改造
- 更新登录入口
- 调整会话校验
- 补充错误场景处理

## 执行模式 3：按 Scenario 验收

对于每个实现任务，都回到对应的 spec delta：

1. 找到相关 Requirement
2. 找到其成功与失败 Scenario
3. 验证实现是否覆盖这些行为

## 执行模式 4：文档 / 模板仓库

如果 change 主要修改文档、模板或规范文件，验证方式可以是：

- 模板字段是否完整
- 标题层级是否符合规范
- 路径与文件名是否与仓库约定一致
- 交叉引用是否指向存在的文件

## 常见错误

- 没有按 `tasks.md` 顺序执行，导致依赖混乱
- 只验证代码，不验证 Scenario
- 在实施结束后没有更新 checklist 完成状态
